Github api entry point

Fields§

§octocrab: Octocrab

Octocrab client. This can be trivially cloned

§semaphore: Arc<Semaphore>

A semaphore to control the maximum number of jobs that can be run in parallel

§webhook_url: String

An optional webook for posting to slack, if that feature is enabled

§is_tty: bool

Defines whether or not we’re running in interactive mode

§output: Arc<OnceCell<OutputMode>>

Where output should go. This can only be written to Once

§slack_messages: Arc<Mutex<Vec<String>>>

A message that gets sent to slack at the end of all processing, if it has any contents. This is thread safe.

§slack_errors: Arc<Mutex<Vec<String>>>

An error message that will get sent to slack at the end of all processing, if it has any contents. This field is also thread safe.

pub async fn get_tags( &self, url: &str, ) -> Result<(IndexSet<TagInfo>, HashSet<String>), GitError>

This can be used to fetch tags in a more api-call efficient way than using the rest api. It does mean we have to manually query the graphql endpoint and manually parse the json output, rather than having it done for us by octocrab. We can’t actually get all the information required for an annotated tag, but we can use it to distinguish between lightweight and annotated tags. If we don’t get any annotated tags, we can skip the fairly slow git clone and push process

IndexSet is an implementation of an orderered Set.
